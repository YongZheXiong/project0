## 项目0在迭代一阶段要具备的核心能力

迭代一阶段的目标，不是把项目0功能池中的所有内容全部实现，也不是把所有高级能力一次性做完，而是在当前约束条件下，优先完成平台成立所必需的核心能力，使项目0从“概念设想”转变为“真实可运行、可复现、可扩展、可调试、可演示”的具身智能小车基础平台。迭代一完成后，项目0应至少具备以下核心能力。

第一，项目0应具备**基础自主运动与受控执行能力**。\
也就是说，小车应能够在真实场地中完成前进、后退、转向、停止、恢复等基本运动控制，并在导航和任务执行过程中保持基本可控状态，而不是仅能由人工遥控或只在理想条件下短时运行。这一能力是后续导航、语义任务执行、安全约束和实验验证的最底层前提。

第二，项目0应具备**面向真实走廊环境的基础环境感知能力**。\
小车应能够感知空间结构、可通行区域、边界、静态与动态障碍物、高风险区域、低光照条件以及部分地形风险，并据此支撑路径规划、安全约束和状态判断。这里的感知能力不要求在迭代一阶段做到高度复杂的语义理解或最优鲁棒性，但必须达到“能支撑真实场景稳定运行”的程度。

第三，项目0应具备**基础定位与建图能力**。\
小车应能够在教学楼单层走廊、回字形通路等真实场景中完成初始定位、实时定位、位姿估计、地图构建、地图更新、回环处理和一定程度的位姿校正，从而为持续导航提供稳定的空间参照。迭代一不要求在这一步就把长走廊鲁棒性做到研究级，但必须建立一个真实可用的SLAM与定位建图底座。

第四，项目0应具备**基础导航与避障通行能力**。\
小车应能够根据目标位置或目标区域生成路径，进行路径跟踪、局部避障、重规划和到达判定，并在走廊环境中完成基本自主导航。这里的重点不是炫技，而是让系统真正形成从“知道自己在哪”到“知道该往哪去”再到“真正走过去”的主闭环。

第五，项目0应具备**基础语义导航能力**。\
小车应能够接收基础语义目标或区域语义信息，并将这些信息转换为可执行的导航目标，形成语义输入、目标关联、导航执行和结果反馈的基础链路。迭代一阶段的语义导航重点在于“成立”，即形成初步的具身智能上层任务能力，而不是一次性展开完整复杂的语义感知与语义地图系统。

第六，项目0应具备**基础任务决策与执行调度能力**。\
小车不应只是被动执行孤立控制指令，而应能够根据任务目标、任务约束、当前状态和异常情况进行基础行为切换、任务级策略调整、执行过程监控与结果反馈。也就是说，项目0在迭代一阶段就要初步具备“系统在做任务”而不是“模块在各自运行”的特征。

第七，项目0应具备**安全运行与人工接管能力**。\
考虑到真实场地中存在楼梯口、天井边缘、电梯口、障碍物、夜间运行和通信/供电异常等高风险因素，项目0必须在迭代一阶段建立安全底线。系统应能够对危险区域进行约束，对异常状态进行停车和保护，并具备人工紧急接管、人工恢复和人工控制权切换能力。这一能力不是附属能力，而是平台成立的硬条件。

第八，项目0应具备**系统级运行管理能力**。\
小车应具备系统启动、关闭、模式管理、状态管理、异常分级、参数配置、自检和运行监控等系统级管理能力，使项目0具备平台雏形，而不是单纯的一条导航链路。只有具备这些能力，后续迭代、测试、部署、演示和答辩表达才有真正稳定的基础。

第九，项目0应具备**数据记录、调试、测试与复盘能力**。\
小车应能够记录原始数据、过程数据、结果数据和日志，并具备调试支撑、测试支撑、实验观测、失败复盘和展示素材采集能力。这一能力在迭代一阶段就必须成立，因为它不仅服务于工程落地，也直接决定迭代二能否进行有效实验设计和研究验证。

第十，项目0应具备**真实部署与后续扩展支撑能力**。\
小车应具备传感器管理、通信管理、电源管理、补光协同、部署迁移、版本管理和后续扩展接口，使其在完成迭代一后不仅“当前能用”，而且“后面还能继续增强”。换言之，迭代一完成后，项目0应已经具备研究平台的基本属性，而不是只能停在第一次演示。

综合而言，迭代一完成后，项目0应不只是“一辆能跑的小车”，而应成为一个已经具备**基础闭环能力、基础智能任务能力、安全底线能力、系统管理能力和实验支撑能力**的具身智能小车基础平台。这个平台的定位不是终态，而是后续迭代二进行技术增强和科研实验的稳定起点。
