# 项目0最终系统定位

## 项目0最终要形成的，不是一个仅用于演示单一功能的普通小车，也不是一个停留在仿真层面的概念验证系统，而是一个**面向教学楼室内/半开放走廊场景、具备感知—决策—执行闭环、能够真实运行、可复现、可扩展、可继续研究的具身智能地面移动平台**。该平台以真实物理实体系统为载体，以工程落地为主，以研究延展为辅，整体定位为“**类产品级基础平台 +后续科研实验载体**”。

## 该系统的核心任务场景，不是泛化到所有环境，也不是追求脱离约束的通用机器人能力，而是**围绕教学楼单层长走廊、回字形通路、高风险边界区域、夜间不稳定光照条件等真实环境约束，稳定实现基础运动控制、SLAM导航与语义导航，并在此基础上保留后续技术升级与科研实验空间**。因此，项目0从一开始就不是“竞赛小作品”或“一次性演示品”的定位，而是一个能够持续迭代、能够承载实验、能够沉淀报告与答辩表达的系统化平台。

## 从项目形态上看，项目0应同时具备四重属性。

- 1.  它是一个**真实运行的实体系统**。项目最终验收不能依赖纯仿真或“仿真加讲解”的替代方案，而必须以真机闭环验证为准。仿真可以作为前期辅助验证手段，但系统的成立标准、能力成立标准和结果表达基础，都必须来源于真实小车平台。

- 2.  它是一个**具备平台属性的工程系统**。项目0不应只关注“能不能跑通一次”，而应具备基础运行管理、安全约束、人工接管、数据记录、调试测试、部署维护和版本演进能力，使其成为一个可复现、可调试、可扩展的平台，而不是一次性拼装作品。

- 3.  它是一个**面向自动化/人工智能交叉视角的具身智能系统**。该系统既要体现控制、导航、系统稳定性和真实场景闭环能力，也要体现语义导航、智能任务理解、多传感器协同等AI方向特征。也就是说，项目0的定位不是单纯控制系统，也不是单纯视觉AI系统，而是一个“**控制为骨架，AI为上层能力，系统集成为核心价值**”的具身智能平台。

- 4.  它是一个**后续科研实验的载体系统**。项目0在迭代一阶段先完成平台成立，在迭代二阶段围绕重点能力进行增强和实验验证。因此，项目0从整体定位上就必须预留研究延展空间，不能把系统做成结构封闭、接口僵硬、无法继续增强的终态产品。它应当是一个“先做成，再做强，再做深”的平台。

## 综合而言，项目0最终系统定位可以归纳为一句话：**项目0是一个面向教学楼走廊真实场景、以真机闭环为验收标准、以控制与导航稳定性为底座、以语义导航等AI能力为上层扩展、兼具工程平台属性与科研实验承载能力的具身智能地面移动平台。**
