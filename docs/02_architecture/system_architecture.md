# system_architecture

---

## 1. 文档定位

本文档定义项目0的**系统级骨架**，只回答两个问题 ：

1. **系统由什么组成**：有哪些一级模块、二级模块
2. **模块之间是什么关系**：依赖关系、信息流方向、控制流方向、主链路与辅助链路

本文档**不定义具体实现方式**，以下内容**不属于本文件范围** ：

- 不定义具体接线方式、电源电压、线径、接口型号
- 不定义具体 ROS2 话题、服务、动作名称
- 不定义具体算法参数、控制参数、标定参数
- 不定义节点级部署细节与进程级实现细节
- 不定义具体计算节点职责划分（由计算与通信架构承接）
- 不定义具体软件包划分（由软件架构承接）

本文档的作用：

- 作为项目0全部子架构的**上位骨架**
- 为后续计算与通信架构、硬件架构、电源架构、软件架构、接口定义等文档划定边界
- 保证项目0始终围绕"真实具身智能地面移动平台"这一定位推进

---

## 2. 系统总目标与架构约束

项目0最终要形成的是一个面向教学楼单层长走廊/回字形通路/高风险边界区域的、具备"感知—决策—执行"闭环的真实具身智能地面移动平台。迭代一目标是先形成最小闭环基础平台，迭代二再做重点能力增强。

### 2.1 聚焦主闭环

系统主闭环围绕：

**底盘执行 → 环境感知 → 空间定位 → 自主导航 → 任务执行**

展开，形成真实可运行的主链路，而不是并列堆叠多个松散能力。

### 2.2 安全、管理、记录不是附属件

项目0运行于存在楼梯口、天井边缘、电梯口、排水槽、夜间弱光等风险的真实环境，因此安全约束、系统管理、数据记录必须作为辅助骨架链路长期并存，而不是后期补丁。

### 2.3 架构必须支撑两阶段演进

架构既要满足迭代一"平台成立"，也要为迭代二"定位建图鲁棒性增强、多传感器融合增强、语义导航深化"预留位置，因此模块边界不能做死。

### 2.4 架构必须适配单人开发

由于项目0由单人完成，系统总架构必须层次清晰、主次明确、边界稳定，避免在总架构层面引入过多非主任务方向内容。

## 3. 系统总架构分层

项目0系统总架构采用自下而上的分层结构，划分为**五个主链路层级**与**三个贯穿辅助链路**。

---

### 3.1 执行层

执行层是系统与物理世界直接发生作用的最底层，负责把上层控制结果转化为真实运动行为。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 运动执行模块 | 接收速度指令，驱动电机执行真实运动，包括前进、后退、转向与停车 |
| 底层状态反馈模块 | 读取编码器与底层传感器，向上反馈轮速、里程计与底层异常状态 |

> 执行层依赖底盘物理结构（车体、驱动轮、从动轮等），具体由硬件架构文档定义。

**系统角色：接受控制命令，输出真实运动，并向上反馈底盘状态。**

---

### 3.2 感知层

感知层负责从真实环境中获取系统运行所需的外部信息与部分内部状态信息，为定位、导航、安全判断提供输入。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 空间感知模块 | 获取环境结构信息、可通行区域信息和障碍物信息 |
| 视觉/深度感知模块 | 获取场景RGB图像和深度信息，支撑弱光条件下的感知辅助 |
| 姿态与运动感知模块 | 获取惯性测量数据和运动变化信息 |
| 风险环境感知辅助模块 | 在高风险区域和弱光条件下提供感知支撑信息 |

**系统角色：持续获取环境与自身运动信息，为空间认知、导航、安全提供感知输入。**

---

### 3.3 空间认知层

空间认知层负责在感知输入基础上建立和维护系统对自身位置与环境结构的理解。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 建图模块 | 在走廊/回字形环境中构建可用于导航的环境地图 |
| 定位模块 | 在已有地图上持续估计系统当前位置，包括初始定位和重定位 |
| 地图管理模块 | 负责地图的保存、加载、版本区分等持久化管理 |

**系统角色：形成系统的空间状态基础，为导航与任务执行提供"我在哪、环境是什么样"的认知。**

---

### 3.4 行动规划层

行动规划层负责根据导航目标和当前空间状态，规划可行路径并控制运动执行。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 全局路径规划模块 | 根据目标点在全局地图上规划可行路径 |
| 局部避障与路径跟踪模块 | 根据实时感知数据进行局部避障和路径跟踪控制 |
| 代价地图管理模块 | 维护全局/局部代价地图，将障碍物、禁区等约束体现在代价中 |
| 导航目标管理模块 | 接收、管理和跟踪导航目标，判断可达性，输出到达/失败状态 |
| 导航异常处理模块 | 检测导航卡死、路径失效等异常，触发重规划或安全停车 |

**系统角色：负责"怎么去"，将空间认知结果转化为安全可行的运动路径并控制执行。**

---

### 3.5 任务决策层

任务决策层是系统上层统一入口，负责将语义级目标转化为导航级行为，并管理任务生命周期。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 语义导航模块 | 维护语义位置定义，接收语义指令，将语义目标转换为导航目标 |
| 任务管理模块 | 管理任务接收、解析、生命周期状态和异常处理 |
| 行为模式管理模块 | 管理系统行为模式（待机/手动/自主/急停）的切换与仲裁 |

**系统角色：负责"做什么"，将上层意图组织为系统可执行的行为序列。**

## 4. 辅助链路定义

除主链路外，项目0还具有三个长期并行的辅助链路。它们不替代主链路，但对主链路形成约束、支撑与留痕。

---

### 4.1 安全保障链路

安全链路贯穿所有层，对系统运行进行实时约束和保护。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 风险约束模块 | 将高风险区域和禁行区域的约束体现在系统行为中 |
| 异常监测模块 | 对系统各层运行异常进行检测 |
| 安全处置模块 | 在异常或危险状态下执行停车、降级、任务中止等保护动作 |
| 人工接管模块 | 支撑人工急停、人工控制权接管和人工恢复授权 |
| 安全事件输出模块 | 将安全事件上报并留痕 |

**系统角色：保证系统在异常、危险或不可控状态下优先进入安全状态。安全链路具有全系统最高优先级。**

---

### 4.2 系统管理链路

系统管理链路负责保障系统整体可启动、可关闭、可监控、可配置 [14]。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 启停管理模块 | 管理系统启动流程、关闭流程和模式化启动 |
| 系统状态管理模块 | 管理各模块在线状态、系统运行状态和异常等级 |
| 自检模块 | 运行前检查关键能力可用性，决定是否允许进入自主运行 |
| 配置管理模块 | 支撑系统参数配置和版本配置管理 |
| 运行监控模块 | 监控系统关键模块运行状态，提供基础运行可观测能力 |

**系统角色：保证项目0作为平台运行，而不是一次性拼接起来的功能链路。系统管理具有次高优先级（仅低于安全）。**

---

### 4.3 数据记录与复盘链路

记录链路负责对运行过程、结果、异常和实验数据进行留痕。

包含二级模块：

| 二级模块 | 职责概述 |
|----------|---------|
| 原始数据记录模块 | 记录传感器、底盘反馈、空间状态等原始数据 |
| 过程记录模块 | 记录导航过程、任务状态、模式切换、安全事件等运行过程 |
| 结果记录模块 | 记录任务结果、运行结果和性能结果 |
| 调试与复盘支撑模块 | 支撑问题定位、运行回放和实验复现 |

**系统角色：让系统运行可观察、问题可追溯、实验可复现、成果可表达。记录链路不参与控制，不参与任务组织，但必须覆盖主链路与辅助链路全过程。**

## 5. 一级模块清单

总架构层面只保留以下八个一级模块。后续任何子架构、模块设计、接口设计，都应归属于这八个一级模块之一，不在总架构层面额外膨胀新的平行主模块。

### 5.1 主链路一级模块

1. 执行模块
2. 感知模块
3. 空间认知模块
4. 行动规划模块
5. 任务决策模块

### 5.2 辅助链路一级模块

6. 安全保障模块
7. 系统管理模块
8. 数据记录与复盘模块

## 6. 主链路定义

项目0的系统主链路为 ：

**执行模块 → 感知模块 → 空间认知模块 → 行动规划模块 → 任务决策模块**

其运行逻辑为闭环：

1. **执行模块**提供真实运动能力与运动反馈
2. **感知模块**持续获取环境与运动相关输入
3. **空间认知模块**形成当前位置与环境地图
4. **行动规划模块**根据目标生成路径与通行决策
5. **任务决策模块**组织语义目标、任务状态与导航行为
6. 最终再由任务/导航结果回到执行模块，驱动真实移动完成任务

因此主链路本质上是闭环：

**任务目标下发 → 导航决策 → 底盘执行 → 感知更新 → 定位更新 → 导航修正 → 任务反馈**

这个闭环结构正对应项目0"感知—决策—执行"系统定位 。

## 7. 模块间关系与控制优先级

### 7.1 主链路依赖关系

| 关系 | 说明 |
|------|------|
| 执行模块是全系统物理基础 | 感知、建图、定位、导航都依赖真实底盘运行，没有执行层系统无法形成真机闭环 |
| 感知模块是空间认知模块的前提 | 定位与建图依赖感知输入 |
| 空间认知模块是行动规划模块的前提 | 导航必须建立在可用定位与可用地图基础之上 |
| 行动规划模块是任务决策模块的执行基础 | 任务层通过导航层实现空间移动，任务层负责"做什么"，导航层负责"怎么去" |
| 任务决策模块是系统上层统一入口 | 语义输入、目标组织、任务状态流转在该层统一发生 |

### 7.2 辅助链路与主链路的关系

| 关系 | 说明 |
|------|------|
| 安全链路依赖全系统状态输入 | 需要来自感知、定位、导航、执行和任务的状态信息 |
| 安全链路对主链路具有最高优先级中断权 | 可中断任务执行、可中断导航执行、可直接要求执行模块进入安全状态 |
| 管理链路依赖各模块可观测状态 | 需要各模块输出运行状态、自检结果、异常等级 |
| 管理链路具有准入与组织权 | 决定系统是否允许进入某种运行模式，决定是否允许自主任务开始 |
| 记录链路依赖全链路数据输出 | 必须覆盖执行、感知、定位、导航、任务、安全、管理全过程 |
| 记录链路不参与优先级竞争 | 全程伴随，不干预控制和任务 |

### 7.3 控制优先级总表

| 优先级 | 模块 | 说明 |
|--------|------|------|
| 最高 | 安全保障模块 | 可中断一切业务执行 |
| 次高 | 系统管理模块 | 决定系统运行准入 |
| 业务执行 | 任务决策 > 行动规划 > 执行 | 任务决定"做什么"→导航决定"怎么走"→执行"实际去做" |
| 状态支撑 | 空间认知 > 感知 | 感知提供原始输入，空间认知形成状态基础 |
| 全程伴随 | 数据记录与复盘 | 不竞争优先级，始终跟随 |

## 8. 一级模块关系表

| 一级模块 | 主要职责 | 直接上游 | 直接下游 | 所属链路 |
|----------|---------|---------|---------|---------|
| 执行模块 | 真实运动执行与底层状态反馈 | 行动规划模块（接收速度指令） | 感知模块（物理运动产生可感知变化） | 主链路 |
| 感知模块 | 环境与运动信息获取 | 执行模块（运动产生新场景） | 空间认知模块（提供感知输入） | 主链路 |
| 空间认知模块 | 建图、定位、地图管理 | 感知模块（感知输入） | 行动规划模块（提供位置与地图） | 主链路 |
| 行动规划模块 | 路径规划、避障、导航控制 | 空间认知模块（位置与地图）、任务决策模块（导航目标） | 执行模块（输出速度指令） | 主链路 |
| 任务决策模块 | 语义导航、任务管理、行为模式管理 | 外部输入（语义指令/任务请求） | 行动规划模块（输出导航目标） | 主链路 |
| 安全保障模块 | 风险约束、异常监测、安全处置、人工接管 | 全系统状态输入 | 可中断主链路任意模块 | 辅助链路 |
| 系统管理模块 | 启停管理、状态管理、自检、配置、监控 | 全系统模块状态 | 控制系统运行准入 | 辅助链路 |
| 数据记录与复盘模块 | 数据记录、过程记录、调试复盘支撑 | 全系统数据输出 | 不输出控制信号 | 辅助链路 |

## 9. 后续子架构归属

本文档确定了总架构骨架后，以下子架构各自回答不同层面的问题，均挂靠在本总架构之下：

| 子架构 | 回答的问题 | 与总架构的关系 |
|--------|-----------|---------------|
| 计算与通信架构 | Orin NX 与 STM32 各自负责什么、之间怎么通信、ROS2 节点怎么划分 | 是总架构中各模块在计算平台上的落位 |
| 软件架构 | ROS2 workspace结构、包划分、话题/服务/动作接口、数据流与控制流 | 是总架构中各模块在软件层面的实现组织 |
| 硬件架构 | 传感器安装位置、底盘结构布局、接线方案 | 是总架构中执行层和感知层的物理落位 |
| 电源架构 | 电池→各设备供电路径、电压转换方案、电源保护 | 是总架构全系统的能量供给基础 |
| 接口定义 | 模块间具体的话题/服务/动作名称、数据类型、调用关系 | 是总架构中"模块间关系"的进一步形式化 |

## 10. 迭代视角下的架构稳定性

### 10.1 迭代一中的作用

在迭代一阶段，系统总架构支撑最小闭环正式成立：

- 执行层成立
- 感知层成立
- 空间认知层成立
- 行动规划层成立
- 任务决策层成立
- 安全、管理、记录三条辅助链路成立

### 10.2 迭代二中的作用

在迭代二阶段，系统总架构本身不变，重点变化发生在层内能力增强：

- **迭代二前段**（优先推进）：空间认知层增强定位建图鲁棒性；感知层与空间认知层之间增强多传感器融合（将编码器里程计、D435i视觉/深度信息融入定位系统，外挂回环检测与退化处理）
- **迭代二后段**（在前段基本完成后推进）：任务决策层与行动规划层之间深化语义导航能力 

迭代二是在既有总架构骨架上的增强，不是重新换骨架。

项目0系统总架构可以概括为：

> 项目0采用"执行层—感知层—空间认知层—行动规划层—任务决策层"的五层主链路骨架，并由"安全保障链路—系统管理链路—数据记录与复盘链路"三条辅助链路贯穿支撑，形成一个面向真实走廊场景、以真机闭环为验收标准、具备平台属性与后续研究扩展能力的具身智能地面移动平台总架构。
