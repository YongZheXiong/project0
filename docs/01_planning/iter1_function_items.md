# 迭代一十项核心能力展开为中粒度功能条目（定稿版）

---

## 一、文档定位说明

本文档用于完成 P1 阶段中的第一个核心任务：
将项目0在迭代一阶段必须具备的十项核心能力，系统展开为可继续进入"范围裁剪、技术选型、架构设计、WBS拆解"的中粒度功能条目清单。

本文档的作用不是直接给出代码实现，也不是在此阶段直接做底层技术细节，而是形成后续以下工作的统一输入基础：

1. 区分各功能条目属于：
   - 迭代一必须实现
   - 迭代一接口预留
   - 不纳入项目0
2. 为后续《迭代一功能清单与技术选型表》提供功能展开底稿
3. 为系统架构设计、模块划分、节点规划、接口定义、WBS拆解提供基础
4. 保证功能规划严格贴合项目0的真实场景约束与平台定位，而不是泛化空想

---

## 二、展开原则

### 2.1 粒度原则

这里的"中粒度功能条目"不等于抽象能力口号，也不等于最终代码级任务。
它处在"核心能力"与"具体实现任务"之间，要求满足以下标准：

- 足够具体，能明确系统到底要具备什么功能
- 足够稳定，不因未来技术方案调整而失效
- 足够可拆，后续可以继续分解为模块、接口、测试项和Sprint任务
- 足够贴合场景，明确服务于教学楼单层回字形走廊真实运行

### 2.2 约束原则

所有功能条目的展开必须服从以下项目总约束：

- 真机主线，仿真辅助，真机验收
- 单人 solo 可推进、可联调、可回退
- 以教学楼单层室内/半开放回字形走廊为核心场景
- 必须显式考虑楼梯口、天井边缘、电梯口、排水槽、局部落差、夜间弱光等高风险因素
- 项目0定位是"类产品级基础平台 + 后续科研实验载体"
- 迭代一目标是平台成立，不是一次性把所有高阶能力做满
- 工程与学术权重约为 7:3，先做成，再做强，再做深
- 必须保留后续扩展、实验验证、报告表达与复盘支撑能力

### 2.3 输出边界

本文档只展开功能条目，不在本轮中处理以下内容：

- 不做技术方案定稿
- 不做算法框架最终选型
- 不做器件型号最终定稿
- 不做节点/话题/协议细节定稿
- 不做WBS和Sprint排期细化

## 三、迭代一十项核心能力——中粒度功能条目展开

---

### 1. 底盘运动控制能力

#### 1.1 基础运动执行能力

1. 系统能够接收上层下发的速度指令并转化为电机控制信号
2. 系统能够执行前进、后退、原地转向、弧线转弯等基础运动
3. 系统能够按照指定线速度与角速度进行运动
4. 系统能够在指令为零时可靠停车
5. 系统能够在无指令输入超时后自动停车

#### 1.2 电机闭环控制能力

1. 系统能够对驱动电机进行速度闭环控制
2. 系统能够读取电机编码器反馈并计算实际转速
3. 系统能够根据编码器反馈进行PID调节
4. 系统能够在不同负载和地面条件下保持基本速度跟踪能力

#### 1.3 里程计能力

1. 系统能够基于编码器数据计算轮式里程计
2. 系统能够输出里程计位姿估计
3. 系统能够以稳定频率发布里程计数据供上层使用

#### 1.4 底层通信能力

1. 系统具备上位机与底层控制板之间的双向通信能力
2. 系统具备通信协议定义与数据帧格式约定
3. 系统具备通信异常检测能力（超时、丢包、校验失败）
4. 系统具备通信恢复或安全降级基础能力

#### 1.5 底盘安全保护能力

1. 系统具备电机过流/过温基础保护能力
2. 系统具备速度限幅能力
3. 系统具备急停指令响应能力（接收到急停信号后立即停止电机）
4. 系统具备通信中断后的自动停车保护能力

---

### 2. 环境感知能力

#### 2.1 激光雷达感知能力

1. 系统能够接入激光雷达并获取点云或扫描数据
2. 系统能够以稳定频率发布激光雷达数据供建图、定位、避障使用
3. 系统能够对激光雷达数据进行基础有效性检查

#### 2.2 视觉感知能力

1. 系统能够接入深度相机并获取RGB图像与深度图像
2. 系统能够以稳定频率发布视觉数据
3. 系统能够对视觉数据进行基础有效性检查
4. 系统能够在弱光条件下借助补光支撑维持基本视觉感知能力

#### 2.3 IMU感知能力

1. 系统能够接入IMU并获取加速度与角速度数据
2. 系统能够以稳定频率发布IMU数据
3. 系统能够对IMU数据进行基础有效性检查

#### 2.4 感知数据基础管理能力

1. 系统具备各传感器数据的时间戳管理能力
2. 系统具备各传感器数据的坐标系定义与管理能力
3. 系统具备传感器在线/离线状态基础检测能力
4. 系统具备感知数据异常的基础告警能力

#### 2.5 高风险区域感知辅助能力

1. 系统能够在楼梯口、天井边缘、电梯口等高风险区域附近提供感知数据
2. 系统能够为后续安全模块提供高风险区域相关的感知输入
3. 系统能够在弱光/夜间条件下维持基本感知能力（配合补光）

---

### 3. 定位与建图能力

#### 3.1 建图能力

1. 系统能够在教学楼走廊/回字形通路场景中进行实时建图
2. 系统能够输出可用于后续定位和导航的地图
3. 系统能够在建图过程中处理走廊长直结构和回字形重复结构
4. 系统能够在建图过程中处理半开放区域（天井侧、连廊侧）
5. 系统能够保存建图结果供后续使用

#### 3.2 地图持久化与管理能力

1. 系统能够将建图结果保存为持久化地图文件
2. 系统能够在启动时加载已有地图文件
3. 系统能够支持地图文件的版本区分与基础管理
4. 系统能够在不同地图之间进行切换（如不同楼层或不同建图版本）

#### 3.3 定位能力

1. 系统能够在已有地图上进行实时定位
2. 系统能够在启动时完成初始定位（在已有地图上确定当前位置）
3. 系统能够在运行过程中保持持续定位
4. 系统能够输出定位结果供导航和任务模块使用
5. 系统能够在定位质量下降时输出告警或状态标识

#### 3.4 重定位能力

1. 系统能够在定位丢失后尝试重定位
2. 系统能够在重定位失败时输出告警并触发安全保护

#### 3.5 定位建图基础性能可观测能力

1. 系统能够输出定位状态（正常/异常/丢失）
2. 系统能够输出建图状态（建图中/建图完成/建图异常）
3. 系统能够为后续实验记录提供定位精度相关的基础数据

---

### 4. 自主导航与避障能力

#### 4.1 全局路径规划能力

1. 系统能够在已有地图上根据目标点进行全局路径规划
2. 系统能够输出从当前位置到目标位置的可行路径
3. 系统能够在路径规划时考虑地图中的障碍物与禁区
4. 系统能够在路径被阻断时触发重规划

#### 4.2 局部避障与路径跟踪能力

1. 系统能够在导航执行过程中进行局部避障
2. 系统能够根据实时感知数据调整局部运动轨迹
3. 系统能够跟踪全局路径并执行运动控制
4. 系统能够在遇到无法绕行的障碍物时安全停车

#### 4.3 导航目标管理能力

1. 系统能够接收导航目标点（坐标形式）
2. 系统能够判断目标点是否可达
3. 系统能够在到达目标点后输出到达确认
4. 系统能够在导航失败后输出失败状态
5. 系统能够支持连续多目标点依次导航

#### 4.4 代价地图管理能力

1. 系统能够维护全局代价地图
2. 系统能够维护局部代价地图
3. 系统能够将高风险区域标注为禁区或高代价区域
4. 系统能够将实时感知到的障碍物更新到代价地图中

#### 4.5 高风险区域导航约束能力

1. 系统不会规划出经过楼梯口、天井边缘、电梯口等高风险区域的路径
2. 系统能够在接近高风险区域时触发减速或停止
3. 系统能够将高风险区域约束持久化到地图或代价地图中

#### 4.6 导航异常处理能力

1. 系统能够检测导航卡死（长时间未移动或无法前进）
2. 系统能够在导航卡死时触发安全停车
3. 系统能够在导航失败时向上层输出异常状态
4. 系统能够在导航过程中响应急停指令

#### 4.7 导航过程数据输出能力

1. 系统能够输出当前导航路径
2. 系统能够输出当前导航速度
3. 系统能够输出导航执行过程中的关键事件（到达、重规划、避障触发、失败等）

---

### 5. 基础语义导航能力

> **边界说明**：能力5聚焦于"语义输入→语义理解→导航目标转换→任务结果反馈"这一段。导航执行本身（路径规划、避障、运动控制）归属能力4。能力5调用能力4的导航接口，不重复展开导航执行细节。

#### 5.1 语义位置定义与管理能力

1. 系统能够定义场景中的语义位置或语义区域（如"厕所门口""楼梯口A""电梯口"等）
2. 系统能够维护语义标签与地图坐标/区域的映射关系
3. 系统能够对语义位置表进行增删改查
4. 系统能够持久化语义位置表并在启动时加载

#### 5.2 语义输入接收与解析能力

1. 系统能够接收语义级导航指令（如"去厕所"）
2. 系统能够解析语义指令并提取目标语义标签
3. 系统能够查找语义标签对应的地图坐标或区域
4. 系统能够对不在当前语义位置表中的目标进行无效处理与反馈

#### 5.3 语义→导航目标转换能力

1. 系统能够将语义目标转换为能力4可接收的导航目标点
2. 系统能够将转换后的导航任务提交给导航模块执行
3. 系统能够跟踪该导航任务的执行状态

#### 5.4 语义任务结果反馈能力

1. 系统能够反馈"已接收任务"
2. 系统能够反馈"正在执行"
3. 系统能够反馈"到达目标"
4. 系统能够反馈"目标无效/未找到"
5. 系统能够反馈"导航失败/任务中止"
6. 系统能够提供基础可观测的语义任务状态输出

#### 5.5 语义导航边界控制能力

1. 语义导航能力应基于已定义语义位置/区域，不扩展到开放式世界理解
2. 语义导航应优先保证"链路成立"，而不是追求复杂语义推理
3. 语义导航在迭代一中以基础语义闭环成立为目标，为迭代二深化预留空间

---

### 6. 基础任务决策与执行调度能力

#### 6.1 任务接收与解析能力

1. 系统能够接收任务级请求（区别于底层运动控制指令）
2. 系统能够将输入请求解析为结构化任务对象
3. 系统能够区分单步动作（如"去某点"）与多步骤顺序任务（如"去A→等待→去B"）
4. 系统能够对不合法或不可执行的任务请求进行拒绝与反馈

#### 6.2 任务生命周期管理能力

1. 系统能够管理任务状态：未开始→执行中→成功→失败→被取消
2. 系统能够在任务执行过程中持续跟踪状态变化
3. 系统能够在任务结束后输出最终结果状态
4. 系统能够记录任务执行过程信息（启动时间、结束时间、结果、异常原因）

#### 6.3 任务与导航目标的绑定能力

1. 系统能够将任务目标绑定到具体导航目标点
2. 系统能够在任务执行时调用导航模块执行对应导航
3. 系统能够在导航完成/失败时更新任务状态

#### 6.4 多步骤顺序任务基础串联能力

1. 系统能够接收包含多个步骤的顺序任务
2. 系统能够按步骤依次执行（前一步完成后自动进入下一步）
3. 系统能够在某一步骤失败时终止整个任务并反馈失败步骤

#### 6.5 任务取消与中断处理能力

1. 系统能够响应外部发出的任务取消请求
2. 系统能够在安全机制触发时中断当前任务
3. 系统能够在任务被取消或中断后将系统置于安全状态
4. 系统能够记录任务取消/中断的原因和时间

#### 6.6 任务失败恢复与安全停止策略

1. 系统能够在导航失败时选择重试或终止任务
2. 系统能够在定位异常时暂停任务执行
3. 系统能够在安全风险出现时中止当前任务并安全停车
4. 系统能够在条件恢复后由人工决定是否恢复执行

#### 6.7 基础行为模式切换能力

1. 系统能够在待机、手动控制、自主导航、急停等行为模式之间切换
2. 系统能够在异常出现时从执行状态切换到保护状态
3. 系统能够在人工接管介入时切换控制权
4. 急停模式具有最高优先级，可随时覆盖其他模式

#### 6.8 多任务基础调度能力

1. 系统能够处理多任务顺序执行的基础需求
2. 系统能够在新任务到来时执行排队、拒绝或覆盖策略
3. 系统能够在接收到更高优先级指令时打断当前任务
4. 迭代一阶段不要求并行多任务执行，只要求顺序调度成立

---

### 7. 安全约束与人工接管能力

#### 7.1 急停能力

1. 系统具备硬件级急停能力（物理急停按钮直接切断电机驱动）
2. 系统具备软件级急停能力（急停指令下发后立即停止运动）
3. 急停响应优先级高于所有其他指令
4. 急停触发后系统进入急停状态，需人工确认后才能恢复

#### 7.2 高风险区域安全约束能力

1. 系统能够在地图或代价地图中标注高风险区域为禁区
2. 系统不会规划经过高风险区域的路径
3. 系统能够在意外接近高风险区域时触发减速或停止
4. 高风险区域约束覆盖楼梯口、天井边缘、电梯口、排水槽、局部落差等

#### 7.3 异常停车能力

1. 系统能够在通信中断时自动停车
2. 系统能够在定位丢失时自动停车
3. 系统能够在感知数据异常时自动停车
4. 系统能够在导航卡死时自动停车
5. 系统能够在电量过低时触发停车保护

#### 7.4 人工接管能力

1. 系统具备人工遥控接管能力
2. 人工接管可随时介入并覆盖自主控制
3. 系统能够在接管介入时安全切换控制权
4. 系统能够记录人工接管触发时间和原因

#### 7.5 速度与运动安全约束能力

1. 系统具备最大速度限幅
2. 系统具备加速度限幅
3. 系统能够在接近障碍物时自动降速
4. 系统能够在窄通道或拐角处降速运行

#### 7.6 安全事件记录能力

1. 系统能够记录所有安全事件（急停、异常停车、接管、禁区触发等）
2. 系统能够记录安全事件的触发时间、触发原因和处理结果
3. 安全事件记录可用于后续复盘和报告

---

### 8. 系统管理与运行支撑能力

#### 8.1 启动与关闭管理能力

1. 系统具备规范化的启动流程（启动顺序管理）
2. 系统具备一键启动或脚本化启动能力
3. 系统具备规范化的关闭流程
4. 系统具备优雅退出能力，避免直接断电式关闭
5. 系统具备建图模式/定位运行模式/任务模式等运行流程的基础组织能力

#### 8.2 模式管理能力

1. 系统具备手动模式
2. 系统具备自主模式
3. 系统具备急停模式
4. 系统具备建图模式与定位运行模式的区分
5. 系统具备模式切换规则与切换条件约束
6. 系统具备模式冲突防护能力

#### 8.3 状态管理能力

1. 系统能够管理整体系统状态
2. 系统能够管理核心模块状态
3. 系统能够管理任务状态、导航状态、定位状态、安全状态
4. 系统能够输出当前系统"可运行/受限运行/不可运行"的基础判断
5. 系统能够表达关键模块在线/离线状态

#### 8.4 自检能力

1. 系统启动时能够检查底层通信是否正常
2. 系统启动时能够检查关键传感器是否在线

## 四、功能条目结构性总结

上述十项核心能力展开后的中粒度功能条目，可归纳为以下五个层面：

| 层面 | 覆盖能力 | 说明 |
|------|----------|------|
| 执行层 | 能力1（底盘运动控制） | 系统最底层的运动执行与硬件控制 |
| 感知与空间层 | 能力2（环境感知）、能力3（定位与建图） | 系统对环境的感知、空间建模与自身定位 |
| 导航与任务层 | 能力4（导航与避障）、能力5（语义导航）、能力6（任务调度） | 系统的自主导航、语义理解与任务管理 |
| 平台管理层 | 能力7（安全约束与人工接管）、能力8（系统管理）、能力9（数据记录与测试复盘） | 系统的安全底线、运行管理与数据支撑 |
| 平台演进层 | 能力10（部署扩展支撑） | 系统的部署、维护与后续扩展基础 |

这意味着，项目0迭代一不是"只做导航"，也不是"只做一辆能动的小车"，而是要完成一个最小但完整的具身智能平台骨架：
能动、能感知、能定位、能导航、能理解基础语义目标、能做基础任务管理、能安全运行、能被接管、能被调试、能被记录、能被复现、能继续扩展。
