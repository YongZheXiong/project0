# 电机驱动模块

## 电驱模块介绍

### 本质
- 连接“控制器”和“电机”的功率执行单元.
### 必要性
在实际系统里，比如单片机、STM32、Arduino、树莓派，这些控制器只能输出很小的电流和较低 功率的控制信号，它们本身不能直接带动电机.原因如下:
- 电机启动时需要较大电流    
- 电机正反转需要切换电流方向
- 调速时需要稳定的功率控制
- 电机工作时会产生反向电动势，可能损坏控制器

## 电驱模块的核心作用

### 功率放大
- 控制器输出的是逻辑电平信号，例如 3.3V 或 5V，但电机可能需要 6V、12V，甚至更高电压和更大电流。驱动模块可以把弱控制信号转换成足够的驱动功率。
### 控制转动方向
- 对于直流电机来说，改变电流方向，电机转动方向就会改变。驱动模块常通过H桥电路实现电流方向切换，从而实现电机正转和反转。
### 调速
- 控制器并不是直接改变电压大小，而是输出一定频率的方波，通过调节高电平占空比，控制电机等效输入功率，进而控制转速。
### 保护控制系统
- 电机属于感性负载，通断时会产生反向电动势。如果没有保护电路，容易烧坏主控芯片。

## 电驱工作链
```
控制器输出控制信号
→ 输入接口/逻辑控制单元理解这些信号是什么意思
→ 驱动控制单元把“控制意图”变成“功率开关指令”
→ 栅极驱动/前级驱动把弱控制信号放大成能推动功率器件的信号
→ 功率驱动级真正切换大电流，把电源送到电机
→ 电机绕组得到电压和电流，产生转矩并转动
→ 检测/保护电路实时监测结果，必要时反过来限制或关断前面各级
```

## 主要组成部分
主要由以下部分组成
- 控制信号接口电路
- 逻辑控制单元
- 驱动控制单元
- 功率驱动级
- 检测与保护

### 控制信号接口电路
这是驱动模块和主控板连接的第一层。它的任务是接收来自 MCU 的控制信号.
例如:
- 方向信号 IN1 / IN2
- 使能信号 EN
- PWM 调速信号
- 步进驱动里的 STEP / DIR
- 无刷驱动里的换相控制信号
- 高级驱动里的 SPI / I2C / CAN 配置接口

### 逻辑控制单元
- 本质: 把外部控制信号翻译成一组内部明确的开关控制规则。
根据输入信号去决定：
- 哪些功率管该导通
- 哪些该关断
- 导通顺序是什么
- 是否允许刹车
- 是否需要死区时间
- 是否检测到异常
### 驱动控制单元
逻辑控制单元虽然已经知道“哪个管该开、哪个管该关”，
但它输出的依然只是小信号。接下来要进入真正的过渡桥梁：驱动控制级。
任务：
- 接收逻辑控制单元生成的时序命令
- 把这个弱逻辑信号“放大成”足以驱动功率器件的电压/电流脉冲
- 快速给功率器件的控制端充放电
- 保证开关动作干净利落
- 防止高低边误导通
- 对高边开关提供特殊驱动条件
如果是 MOSFET，这一层常叫**栅极驱动级**。
如果是更广义的说法，也可以叫**前级驱动**。
### 功率驱动级

#### 本质
- 用某种功率半导体器件，按照控制要求，把电源接到电机上，或者断开，或者换方向
#### 结构
##### 功率器件本体
比如 MOSFET、IGBT、BJT。这是“真正负责接通/断开大电流”的元件。
##### 功率拓扑结构
比如：单管开关,半桥,全桥/H桥,三相桥等等.这决定了功率器件是怎么组合起来给电机供电的。
##### 功率输出对象
**功率驱动级 = 功率器件 + 拓扑连接方式**

### 检测与保护
真实电驱模块还有“反向链路”
- 检测电流
- 检测温度
- 检测电压
- 检测短路/堵转
这些检测结果会反馈给逻辑控制或驱动控制单元。
