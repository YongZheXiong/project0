# DEC-018 人工接管输入方案决策

- 决策状态：确认
- 所属阶段：P1
- 日期：2026-04-19

## 背景

P1 阶段需要明确系统架构与关键实现方案。  
P2 阶段需要支持“手柄或键盘 → cmd_vel → 底层执行”的人工接管与底盘调试路径。

## 内容

确定采用以下人工接管方案：

1. 使用飞智黑武士2游戏手柄；
2. 通过 2.4G USB 接收器接入 Orin NX；
3. 在 Orin NX 上由 teleop / 手动控制节点生成 `cmd_vel`；
4. 通过底盘桥接与 UART 链路将速度指令下发到 STM32。

## 理由

1. 实现路径直接，适合迭代一快速落地。
2. 不需要额外研究 STM32 直接解析游戏手柄协议。
3. 符合单人开发负担可控原则。
4. 能满足 P2 阶段人工接管与调试的需求。

## 影响

1. Orin NX 需要预留 USB 接收器接口与安装空间。
2. 硬件架构中不再采用“传统航模接收机直连 STM32”作为当前实现。
3. 安全上仍必须依赖硬件急停作为最高优先级手段。
