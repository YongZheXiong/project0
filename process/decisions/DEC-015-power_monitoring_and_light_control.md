# DEC-015：电量监测与补光控制

- 状态：确认
- 阶段：P1
- 日期：2026/4/19

## 背景
P1 技术选型中系统管理方案和安全机制方案都需要明确。
系统总架构中，系统管理链路具有运行准入与组织权，记录链路要求覆盖全过程。 
项目场景包含夜间弱光场景，因此补光模块属于缺失硬件与后续能力支撑项

## 决策
确定采用：
1. 迭代一正式实现基础电量监测上报；
2. 主电池电压状态纳入系统管理输入；
3. 补光模块纳入系统控制，不仅仅是手动独立供电；
4. 补光用于支撑弱光场景下的前向视觉辅助感知。

## 理由
1. 系统管理模块需要监控系统关键模块运行状态并决定是否允许进入自主运行。
2. 记录链路要求覆盖全过程，因此电源状态不应完全脱离系统。
3. 夜间弱光是明确场景约束，补光支撑能力应纳入平台设计。
4. 将补光纳入系统控制比纯手动独立方式更有利于后续联动与工程规范性。

## 影响
1. 电源架构需预留电量采样与补光控制路径；
2. 系统管理模块后续需接收电源状态；
3. 后续弱光测试中需验证补光效果。
