# DEC-016：底盘结构与机械实现路线决策

- 状态：确认
- 阶段：P1
- 日期：2026/4/19

## 背景
P1 阶段需要输出硬件架构，内容包括传感器安装位置、底盘结构布局和接线方案。
同时，P2 阶段要完成底盘机械组装，因此必须在 P1 先确定底盘机械实现路线。

## 决策
确定采用以下方案：
1. 底盘使用现成底盘板 / 通用板材方案；
2. 车体采用双层板结构；
3. 双层板之间采用立柱连接；
4. 当前阶段优先目标是“快速装起来”，不优先追求复杂定制机械结构。

## 理由
1. 符合 P1→P2 推进逻辑，先完成可组装、可运行的平台基础。
2. 符合单人开发条件下“先做成，再做强”的原则，降低机械加工复杂度。
3. 有利于缩短从架构设计到真机组装的路径。
4. 便于后续在 P2-P3 阶段继续调整上层板、支架和线束布局。

## 影响
1. 硬件架构采用“下层动力 / 上层计算控制 / 顶部感知”的分层布局。
2. 采购中需要补充底盘板材、立柱、固定件。
3. P2 机械组装阶段可直接按双层底盘实施。
