# DEC-003：迭代一 SLAM 采用 FAST-LIO2（仅 Mid360 + IMU），编码器与 D435 不参与融合

- 状态：确认
- 阶段：P1
- 日期：2026/4/17

## 背景

项目0的 P3 阶段需要完成感知与定位建图能力，要求在走廊场景中实时建图、保存地图、加载地图并进行持续定位。

迭代一功能清单明确：建图和定位采用 FAST-LIO2，并且迭代一的 FAST-LIO2 仅融合 Mid360 + IMU；编码器里程计在迭代一独立发布，不参与 FAST-LIO2 融合；D435 在迭代一独立发布数据，不参与 FAST-LIO2 融合。

计算与通信架构也明确指出，D435 在迭代一中仅用于记录和调试，不参与定位融合。

## 决策

项目0迭代一 SLAM 与定位建图主线采用：

> FAST-LIO2 + Mid360 + IMU。

具体约定：

1. 迭代一只使用 Mid360 点云和 IMU 进入 FAST-LIO2 主链路。
2. 编码器里程计独立发布，但不参与 FAST-LIO2 融合。
3. D435 发布 RGB 和深度数据，但不参与 FAST-LIO2 融合。
4. 编码器融合、D435 视觉/深度融合、回环检测、退化处理等增强能力留到迭代二前段。

## 理由

1. P3 的首要目标是让定位建图能力成立，而不是在迭代一阶段提前展开复杂多传感器融合。
2. 迭代一功能清单已明确 FAST-LIO2 仅使用 Mid360 + IMU，并将编码器融合、视觉融合、回环和退化处理放到迭代二前段。
3. 系统总架构也明确迭代二阶段重点增强空间认知层和感知层之间的多传感器融合，而不是改变总架构骨架。
4. 这种划分能为迭代二保留清晰的增强基线，方便后续做对比实验、结果统计和类论文式报告表达。

## 影响

- 影响阶段：P3、P7、P8。
- 影响模块：p0_sensor_drivers、p0_localization、p0_mapping、p0_data_tools。
- 影响接口：Mid360 点云、IMU、编码器里程计、D435 图像/深度数据的话题组织。
- 影响实验：迭代一 FAST-LIO2 基线测试，迭代二多传感器融合增强对比实验。

## 回退条件（可选）

如果 FAST-LIO2 在目标走廊场景中无法形成可用地图或稳定定位，应优先检查传感器安装、时间同步、TF、参数配置和运行环境；只有在多轮排查后仍不可用时，才重新评估 SLAM 主路线。
