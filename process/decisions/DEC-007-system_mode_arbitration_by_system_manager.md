# DEC-007：模式仲裁权唯一归属 p0_system_manager

- 状态：确认
- 阶段：P1
- 日期：2026/4/17

## 背景

项目0需要支持手动、自主、急停、建图、定位运行等模式，并需要具备切换规则、冲突防护、自检、状态管理和可运行性判断。

系统总架构中，系统管理模块负责启停管理、状态管理、自检、配置和监控，并控制系统运行准入；系统管理链路属于辅助链路，服务于平台稳定运行。

软件架构中，系统管理链路对应 p0_system_manager 与 p0_bringup，模式化启动入口也围绕建图模式、定位导航模式、语义任务模式、手动调试模式和安全模式组织。

## 决策

项目0中系统运行模式的最终仲裁权唯一归属：

> p0_system_manager。

具体约定：

1. p0_system_manager 是系统当前模式与系统状态的事实源。
2. 其他模块可以请求模式切换，但不能各自持有模式仲裁权。
3. p0_bringup 负责按照模式组织 launch 启动入口。
4. 安全相关强制切换需通过安全链路与系统管理链路协同完成。
5. 模式管理不下放给 p0_task_manager 或其他业务模块。

## 理由

1. 迭代一功能清单明确系统管理需要负责手动/自主/急停/建图/定位运行模式、切换规则和冲突防护。
2. 系统总架构明确系统管理模块控制系统运行准入，且系统管理具有仅低于安全的次高优先级。
3. 如果模式仲裁分散到多个模块，会导致任务、导航、安全和手动控制之间出现冲突。
4. 集中到 p0_system_manager 有利于 P6 系统联调、模式化启动、状态可观测和答辩解释。

## 影响

- 影响阶段：P2-P6，尤其 P5 和 P6。
- 影响模块：p0_system_manager、p0_bringup、p0_task_manager、p0_safety_manager、p0_navigation。
- 影响接口：系统模式状态、模式切换请求、自检结果、运行准入状态。
- 影响 launch：mapping、localization、autonomous_task、manual_debug、safe_mode。

## 回退条件（可选）

如果 p0_system_manager 在迭代一实现中过重，应先保留最小模式仲裁功能，不扩展复杂管理功能；不应把模式仲裁拆散到多个业务模块。
