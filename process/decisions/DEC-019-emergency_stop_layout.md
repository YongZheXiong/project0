# DEC-019 急停按钮位置与安全边界决策

- 决策状态：已确认
- 所属阶段：P1
- 日期：2026-04-19

## 背景

安全机制属于 P1 必须明确的技术选型域之一，硬件急停是其中的核心内容。[7][14]  
计算与通信架构已明确：人工急停按钮属于外部输入源，不作为系统内部 ROS2 节点。[18]

## 内容

确定采用以下方案：

1. 急停按钮安装在车体顶部中后部；
2. 按钮允许明显凸起，便于快速按下；
3. 同时加简易保护结构，降低搬运误触风险；
4. 急停通过纯硬件链路切断电机驱动供电，不经过软件链路。

## 理由

1. 顶部中后部位置便于人跟车操作时快速触达。
2. 明显凸起有利于真实场景下的应急操作。
3. 搬运误触风险需要通过简单保护结构平衡。
4. 急停只切动力支路，符合已确定电源架构边界。

## 4. 备选方案与放弃理由

### 方案A：急停安装在侧面
不采用。  
原因：应急可达性通常不如顶部中后部。

### 方案B：急停切断整车总电
不采用。  
原因：不利于保留日志、状态监控和优雅关闭能力。

## 5. 影响

1. 上层板需要预留急停安装孔位与保护结构空间。
2. 电源架构和接线实现必须围绕“只切动力支路”展开。
3. 后续安全测试需要覆盖急停响应与误触风险。

## 6. 后续动作

1. 在 `hardware_architecture.md` 中固化急停位置。
2. 在 `power_architecture.md` 中保持急停切断边界一致。
3. 在测试文档中加入急停功能验证项。
