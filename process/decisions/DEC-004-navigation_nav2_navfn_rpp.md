# DEC-004：迭代一导航框架采用 Nav2 + NavFn + RPP

- 状态：确认
- 阶段：P1
- 日期：2026/4/17

## 背景

项目0的 P4 阶段需要完成自主导航与避障，包括全局路径规划、局部避障、路径跟踪、导航目标管理、代价地图管理、高风险区域约束和导航异常处理。

迭代一功能清单明确：全局路径规划采用 Nav2 + NavFn；局部避障与路径跟踪采用 Nav2 + RPP；代价地图采用 Nav2 Costmap2D，并通过 keepout_filter 实现高风险区域约束。

迭代一到迭代二技术升级路径明确：导航框架保持 Nav2 不变，迭代二可替换插件，例如 RPP 替换为 MPPI，但不推翻导航框架。

## 决策

项目0迭代一导航框架采用：

> Nav2 + NavFn + RPP。

具体约定：

1. Nav2 作为统一导航框架。
2. NavFn 作为迭代一全局路径规划器。
3. RPP 作为迭代一局部路径跟踪与避障控制器。
4. Nav2 Costmap2D 负责全局/局部代价地图。
5. keepout_filter 用于高风险区域禁区或高代价约束。
6. 迭代二如需增强，在 Nav2 框架内替换或扩展插件，不更换导航框架。

## 理由

1. 该方案直接对应迭代一能力4中全局路径规划、局部避障、导航目标管理、代价地图和高风险区域约束的功能要求。
2. Nav2 + NavFn + RPP 能满足迭代一走廊场景的基础自主导航闭环需求。
3. 迭代二升级路径已经明确 Nav2 不变，只做插件替换或增强，符合“不推翻重来”的技术路线。
4. 该方案成熟度高、资料丰富、可调试性强，更适合单人 solo 项目的 P4 工程落地。

## 影响

- 影响阶段：P4、P6、P8。
- 影响模块：p0_navigation、p0_localization、p0_map_manager、p0_safety_manager、p0_system_manager。
- 影响配置：navigation 参数、global_planner、local_planner、costmap、keepout_filter。
- 影响验收：目标点导航、避障、禁区约束、导航失败停车、路径重规划。

## 回退条件（可选）

如果 RPP 在目标场景中跟踪效果不稳定，应优先通过速度限制、加速度限制、代价地图参数和局部规划参数调试解决；若迭代二需要更强局部控制能力，可在 Nav2 框架内评估 MPPI 等插件替换。
